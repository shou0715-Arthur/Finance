# 內部人交易分析系統 規格說明書

> **用途說明**: 本規格說明書作為AI提示語使用，將此完整內容提交給AI，AI將根據規格生成完整的應用程式碼。

## 📋 系統概述

### 系統名稱
【Code Gym】內部人交易分析系統

### 核心功能描述
建立一個基於網頁的內部人交易查詢分析應用，能夠：
1. 查詢指定公司的所有內部人交易記錄（基於股票代碼）
2. 查詢特定內部人在所有公司的交易記錄（基於CIK代碼）
3. 顯示詳細的交易明細，包含交易日期、類型、價格、數量等完整資訊
4. 提供SEC Form 4官方文件連結，方便查證
5. 提供完整的交易類型說明，協助用戶理解不同交易代碼的意義
6. 支援CSV格式匯出，便於進一步分析
7. 顯示交易統計資訊，快速掌握交易概況

### 技術架構要求
- **界面框架**: 使用 Streamlit 框架
- **數據來源**: Financial Modeling Prep API (使用 /stable/ 端點)
- **視覺化工具**: Streamlit 內建表格和圖表元件
- **數據處理**: Pandas, NumPy
- **HTTP請求**: requests
- **日期處理**: datetime
- **部署方式**: 可直接在瀏覽器中運行

### 教育價值定位
- **投資教育**: 幫助學員理解內部人交易的法規和申報流程
- **風險意識**: 培養學員識別不同交易類型的投資參考價值
- **數據素養**: 學習如何解讀SEC Form 4申報資料
- **批判思考**: 理解計劃性變現vs主動交易的差異

### F-002: 數據獲取功能
**功能目標**: 從Financial Modeling Prep API獲取公司資料和內部人交易記錄

**API端點規格**:

**1. 獲取公司基本資料 (get_company_profile)**
- **端點**: `https://financialmodelingprep.com/stable/profile?symbol={symbol}&apikey={api_key}`
- **方法**: GET
- **快取**: `@st.cache_data(ttl=3600)` - 快取1小時
- **參數**: 
  - symbol: 股票代碼
  - apikey: FMP API金鑰
- **超時**: 10秒
- **回傳欄位範例**:
```json
{
  "symbol": "AAPL",
  "price": 232.8,
  "mktCap": 3500823120000,
  "companyName": "Apple Inc.",
  "sector": "Technology"
}
```
- **需要欄位**: 
  - `companyName`: 公司名稱
  - `price`: 當前股價
  - `mktCap`: 市值
  - `sector`: 產業分類
- **錯誤處理**: 
  - 使用 try-except 捕獲異常
  - 回傳None時使用 `st.error()` 顯示錯誤訊息
  - 超時處理

**2. 獲取公司內部人交易 (get_insider_trading_by_symbol)**
- **端點**: `https://financialmodelingprep.com/stable/insider-trading/search?symbol={symbol}&page=0&limit=100&apikey={api_key}`
- **方法**: GET
- **快取**: `@st.cache_data(ttl=1800)` - 快取30分鐘
- **參數**: 
  - symbol: 股票代碼
  - page: 頁碼（固定0）
  - limit: 筆數上限（固定100）
  - apikey: FMP API金鑰
- **超時**: 15秒
- **回傳**: JSON陣列
- **回傳欄位範例**:
```json
{
  "symbol": "AAPL",
  "filingDate": "2025-10-03",
  "transactionDate": "2025-10-02",
  "reportingCik": "0001214156",
  "companyCik": "0000320193",
  "transactionType": "S-Sale",
  "securitiesOwned": 3401863,
  "reportingName": "COOK TIMOTHY D",
  "typeOfOwner": "director, officer: Chief Executive Officer",
  "acquisitionOrDisposition": "D",
  "directOrIndirect": "D",
  "formType": "4",
  "securitiesTransacted": 8395,
  "price": 254.83,
  "securityName": "Common Stock",
  "url": "https://www.sec.gov/Archives/edgar/data/320193/000121415625000011/0001214156-25-000011-index.htm"
}
```
- **需要欄位**: 
  - `symbol`: 股票代碼
  - `transactionDate`: 交易日期
  - `reportingName`: 內部人姓名（大寫格式）
  - `reportingCik`: 內部人CIK代碼
  - `typeOfOwner`: 職位（包含director/officer等完整描述）
  - `transactionType`: 交易類型代碼
  - `securitiesTransacted`: 交易股數
  - `price`: 交易價格
  - `securitiesOwned`: 交易後持股
  - `url`: SEC Form 4連結
  - `filingDate`: 申報日期（選用）
  - `securityName`: 證券名稱（選用）
- **錯誤處理**: 
  - 使用 try-except 捕獲異常
  - 回傳空列表時使用 `st.error()` 顯示錯誤訊息
  - 超時處理

**3. 獲取內部人所有交易 (get_insider_trading_by_cik)**

### F-004: 數據處理與格式化
**功能目標**: 將API回傳的原始數據轉換為易讀的顯示格式

**重要提醒 - API欄位名稱**:
- **Profile API**: 市值欄位為`marketCap`（非`mktCap`）
- **Insider Trading API**: 
  - `reportingName` 為全大寫格式（例如："COOK TIMOTHY D"）
  - `typeOfOwner` 包含完整職位描述（例如："director, officer: Chief Executive Officer"）
  - SEC連結欄位名稱為 `url`（非 `link`）

**數據處理函數 (process_insider_data)**:
```python
def process_insider_data(raw_data):
    """
    處理內部人交易數據
    
    輸入: API回傳的JSON陣列
    輸出: 處理後的DataFrame
    
    處理步驟:
    1. 計算交易金額 = 交易股數 × 交易價格
    2. 提取必要欄位
    3. 按交易日期降冪排序（最新在前）
    """
    if not raw_data:
        return pd.DataFrame()
    
    processed_data = []
    
    for trade in raw_data:
        # 計算交易金額
        transaction_value = trade.get('securitiesTransacted', 0) * trade.get('price', 0)
        
        processed_data.append({
            '交易日期': trade.get('transactionDate', ''),
            '股票代碼': trade.get('symbol', ''),
            '內部人姓名': trade.get('reportingName', ''),
            '內部人代碼': trade.get('reportingCik', ''),
            '職位': trade.get('typeOfOwner', ''),
            '交易類型': trade.get('transactionType', ''),
            '交易股數': trade.get('securitiesTransacted', 0),
            '交易價格': trade.get('price', 0),
            '交易金額': transaction_value,
            '交易後持股': trade.get('securitiesOwned', 0),
            'SEC_URL': trade.get('url', '')
        })
    
    df = pd.DataFrame(processed_data)
    
    # 排序：最新的在前面
    if not df.empty and '交易日期' in df.columns:
        df = df.sort_values('交易日期', ascending=False)
    
    return df
```

**數字格式化函數**:

**1. format_number(num)**:
```python
def format_number(num):
    """格式化數字為千分位"""
    try:
        return f"{int(num):,}"
    except:
        return str(num)
```

**2. format_currency(num)**:
```python
def format_currency(num):
    """格式化為貨幣（美元）"""
    try:
        return f"${num:,.2f}"
    except:
        return str(num)
```

**顯示表格創建函數 (create_transaction_table)**:
```python
def create_transaction_table(df):
    """
    創建顯示用的交易表格
    
    功能:
    1. 複製原始DataFrame避免修改
    2. 格式化數字欄位（千分位、貨幣符號）
    3. 選擇最終顯示的欄位
    
    回傳: 格式化後的DataFrame
    """
    if df.empty:
        return pd.DataFrame()
    
    display_df = df.copy()
    
    # 格式化數字欄位
    display_df['交易股數'] = display_df['交易股數'].apply(format_number)
    display_df['交易價格'] = display_df['交易價格'].apply(format_currency)
    display_df['交易金額'] = display_df['交易金額'].apply(format_currency)
    display_df['交易後持股'] = display_df['交易後持股'].apply(format_number)
    
    # 準備最終顯示的欄位
    final_columns = [
        '交易日期', '股票代碼', '內部人姓名', '內部人代碼', '職位', 
        '交易類型', '交易股數', '交易價格', '交易金額', 
        '交易後持股', 'SEC_URL'
    ]
    
    return display_df[final_columns]
```
