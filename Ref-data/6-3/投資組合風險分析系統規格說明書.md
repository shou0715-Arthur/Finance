# 投資組合風險分析系統 規格說明書

> **用途說明**: 本規格說明書作為AI提示語使用，將此完整內容提交給AI，AI將根據規格生成完整的應用程式碼。

## 📋 系統概述

### 系統名稱
【Code Gym】投資組合風險分析系統

### 核心功能描述
建立一個基於網頁的投資組合風險評估工具，能夠：
1. 提供用戶風險屬性評估問卷，計算風險承受度評分
2. 支援自定義投資組合配置，包含股票和現金部位
3. 獲取投資組合中各項資產的歷史數據和風險指標
4. 計算投資組合整體風險評分和各項風險指標
5. 使用AI進行專業投資組合風險分析和教育性資產配置建議

### 技術架構要求
- **界面框架**: 使用 Streamlit 框架
- **數據來源**: Financial Modeling Prep API
- **AI 模型**: Perplexity API
- **AI 模型版本**: sonar-reasoning
- **視覺化工具**: 互動式圖表 (使用 Plotly Express)
- **數據處理**: Pandas, NumPy
- **HTTP請求**: requests
- **日期處理**: datetime
- **部署方式**: 可直接在瀏覽器中運行

### F-002: 數據獲取功能
**功能目標**: 自動從Financial Modeling Prep API獲取投資組合資產的歷史數據和風險指標
**用戶操作**: 用戶設定投資組合配置後，系統自動取得各項資產相關數據
**數據來源與說明**:
- Financial Modeling Prep API: 獲取股票歷史價格數據和Beta值
- 歷史價格API: `https://financialmodelingprep.com/api/v3/historical-price-full/{symbol}?apikey={api_key}`
- **Exchange Variants API**: `https://financialmodelingprep.com/stable/search-exchange-variants?symbol={symbol}&apikey={api_key}` (用於獲取Beta值和公司基本資訊)

**數據範圍**: 
- 股價數據: 包含歷史收盤價，用於計算波動性
- Beta值數據: 從公司profile中獲取Beta值，用於系統性風險分析
- 成交量數據: 用於風險指標計算
- 公司基本資訊: 包含產業分類和公司名稱
**處理要求**: 
- 自動檢查API響應的有效性
- 提供友善的錯誤提示當API調用失敗時
- 驗證股票代碼格式的正確性
- 處理資料缺失情況

### F-003: 數據處理與計算
**處理目標**: 計算投資組合各項風險指標和綜合風險評分
**計算項目**:
- 個別資產風險評分: 基於歷史波動性和Beta值計算綜合風險分數
- 投資組合整體風險: 根據資產配置比例和個別風險計算加權風險
- 用戶風險承受度評分: 基於問卷結果計算風險屬性評分
- 產業集中度分析: 評估投資組合的產業分散程度
**計算公式**:
- 波動性風險分數: min(年化波動率 * 2.5, 100)
- Beta風險分數: min(Beta值 * 50, 100) 
- 綜合風險分數: (波動性分數 * 70%) + (Beta分數 * 30%)
- 投資組合風險: Σ(個別資產風險 × 配置比例)
**通用品質要求**: 
- 計算結果準確可靠，符合金融風險管理標準
- 妥善處理資料缺失和異常值情況
- 確保風險評分的一致性和可比性
- 實施適當的數據驗證機制
- 保持計算邏輯的可追溯性
