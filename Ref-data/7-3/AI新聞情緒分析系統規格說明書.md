# AI 新聞情緒分析系統 規格說明書

> **用途說明**: 本規格說明書作為AI提示語使用,將此完整內容提交給AI,AI將根據規格生成完整的應用程式碼。

## 📋 系統概述

### 系統名稱
【Code Gym】AI 新聞情緒分析系統

### 核心功能描述
建立一個基於網頁的股票新聞情緒分析工具,能夠:
1. 從Alpha Vantage API獲取指定股票的最新市場新聞和情緒數據
2. 分析新聞來源的分布和情緒傾向評分
3. 使用AI對新聞內容進行深度情緒分析和投資情緒解讀
4. 提供互動式圖表展示新聞情緒分布和趨勢變化

### 技術架構要求
- **界面框架**: 使用 Streamlit 框架
- **數據來源**: Alpha Vantage NEWS_SENTIMENT API
- **AI 模型**: OpenAI
- **AI 模型版本**: gpt-5-mini
- **視覺化工具**: 互動式圖表 (使用 Plotly Graph Objects 和 Plotly Express)
- **數據處理**: Pandas, NumPy
- **HTTP請求**: requests
- **日期處理**: datetime
- **部署方式**: 可直接在瀏覽器中運行

### 重要套件版本要求
- **OpenAI套件**: 使用 openai 版本 1.0.0 或以上的新API格式
- **禁用舊版API**: 絕對不可使用 ChatCompletion.create 舊格式
- **必須使用新格式**: 使用 OpenAI 客戶端初始化方式和 chat.completions.create 方法

### F-002: 數據獲取功能
**功能目標**: 自動從Alpha Vantage NEWS_SENTIMENT API獲取市場新聞和情緒數據
**用戶操作**: 用戶輸入股票代碼後,系統自動取得相關新聞數據

**數據來源API說明**:
- **股價數據API端點**: `https://www.alphavantage.co/query?function=TIME_SERIES_DAILY&symbol={symbol}&apikey={api_key}`
  - API功能: 返回每日股價歷史數據(開盤、最高、最低、收盤、成交量)
  - 用途: 用於在新聞情緒分析頁面中顯示股價走勢圖,讓用戶可以對照新聞情緒與股價表現

- **新聞情緒API端點**: `https://www.alphavantage.co/query?function=NEWS_SENTIMENT&tickers={ticker}&limit={limit}&apikey={api_key}`
  - API功能: 返回實時和歷史市場新聞及情緒數據
  - 支援範圍: 涵蓋股票、加密貨幣、外匯等多種資產類別

**API回傳JSON格式範例**:
```json
{
    "items": "50",
    "sentiment_score_definition": "x <= -0.35: Bearish; -0.35 < x <= -0.15: Somewhat-Bearish; -0.15 < x < 0.15: Neutral; 0.15 <= x < 0.35: Somewhat_Bullish; x >= 0.35: Bullish",
    "relevance_score_definition": "0 < x <= 1, with a higher score indicating higher relevance.",
    "feed": [
        {
            "title": "新聞標題",
            "url": "新聞連結",
            "time_published": "20251004T173200",
            "authors": ["作者名稱"],
            "summary": "新聞摘要",
            "banner_image": "圖片網址",
            "source": "新聞來源",
            "category_within_source": "分類",
            "source_domain": "來源網域",
            "topics": [
                {
                    "topic": "Financial Markets",
                    "relevance_score": "0.108179"
                }
            ],
            "overall_sentiment_score": 0.302077,
            "overall_sentiment_label": "Somewhat-Bullish",
            "ticker_sentiment": [
                {
                    "ticker": "AAPL",
                    "relevance_score": "0.702434",
                    "ticker_sentiment_score": "0.500747",
                    "ticker_sentiment_label": "Bullish"
                }
            ]
        }
    ]
}
```

**數據範圍**: 
- 新聞標題和摘要內容
- 新聞發布時間和來源
- 整體情緒評分和標籤
- 針對特定股票的情緒評分和相關性評分
- 新聞主題分類和相關性評分

**處理要求**: 
- 自動檢查API響應的有效性
- 提供友善的錯誤提示當API調用失敗時
- 驗證股票代碼格式的正確性
- 解析並提取關鍵的情緒數據欄位

### F-003: 數據處理與計算
**處理目標**: 將原始新聞數據轉換為分析用的統計資料

**計算項目**:
- **情緒分布統計**: 計算Bearish、Somewhat-Bearish、Neutral、Somewhat-Bullish、Bullish各類別的新聞數量和佔比
- **平均情緒評分**: 計算所有新聞的平均overall_sentiment_score
- **相關性評分統計**: 計算針對目標股票的平均relevance_score
- **新聞來源分析**: 統計各新聞來源的數量分布
- **時間序列分析**: 按發布時間排序新聞,分析情緒趨勢變化
- **主題分類統計**: 分析新聞涵蓋的主要主題及其分布
- **股價變化計算**: 計算分析期間的股價漲跌幅、波動率
- **情緒股價關聯**: 計算新聞情緒評分與股價變化的時間對應關係

**通用品質要求**: 
- 計算結果準確可靠,符合統計分析專業標準
- 妥善處理資料缺失和異常值情況
- 確保數據時間序列的一致性
- 實施適當的數據驗證機制
- 保持數據處理的可追溯性


### F-004: 主要顯示區域設計
**區域結構**: 使用順序式區域佈局,分為 4 個主要展示區域

#### 區域一:股價趨勢圖
- **標題**: "一、股價趨勢圖"
- **主要圖表**: K線圖 + 移動平均線(MA5、MA10、MA20、MA60)
- **圖表類型**: 使用Plotly Candlestick繪製K線,疊加4條移動平均線
- **顯示數據**: 最近30個交易日的開盤、最高、最低、收盤價格
- **移動平均線配色**: 
  - MA5: 橘色(orange)
  - MA10: 藍色(blue)
  - MA20: 紅色(red)
  - MA60: 紫色(purple)
- **圖表設定**: 
  - 標題顯示股票代碼和日期範圍
  - X軸為日期,Y軸為價格
  - 關閉range slider
  - 圖例水平排列在上方
  - 高度設定為500px
- **數據表格**: 使用expander摺疊方式顯示股價詳細數據表格,包含開盤、最高、最低、收盤、成交量和各MA值

#### 區域二:情緒新聞摘要
- **標題**: "二、情緒新聞摘要"
- **統計總覽區塊**: 
  - 子標題"📊 情緒統計總覽"
  - 使用三欄式佈局(st.columns([1,1,1]))展示:
    - 第一欄: 股票情緒分布(計算各情緒類別數量和百分比)
    - 第二欄: 文章情緒分布(計算各情緒類別數量和百分比)
    - 第三欄: 平均分數(顯示股票情緒、文章情緒、相關性的平均分數)
  - 每個指標使用markdown格式化顯示,用圖示和顏色區分

- **情緒/相關性說明**: 
  - 使用expander摺疊區塊提供詳細說明
  - 情緒分數範圍:-1(極度看空)到+1(極度看多)
  - 五級分類標準:Bullish(≥0.35)、Somewhat-Bullish(0.15~0.35)、Neutral(-0.15~0.15)、Somewhat-Bearish(-0.35~-0.15)、Bearish(≤-0.35)
  - 相關性分數範圍:0(無關)到1(高度相關)
  - 四級分類:高度相關(≥0.8)、相關(0.5~0.8)、中度相關(0.3~0.5)、低度相關(<0.3)

- **新聞列表**: 
  - 取前10則相關性最高的新聞
  - 每則新聞使用expander摺疊顯示
  - expander標題格式:"序號. 📰 新聞標題 | 🎯 相關性標籤"
  - expander內容包含三個欄位:
    - 第一欄:發布時間、股票情緒(圖示+標籤+分數)、文章情緒(圖示+標籤+分數)、相關性(標籤+分數)
    - 第二欄:相關主題(列出前3個topic及其加權分數和標籤)
    - 第三欄:新聞摘要和原文連結

#### 區域三:市場情緒總結
- **標題**: "三、市場情緒總結"
- **功能**: 使用OpenAI分析所有新聞數據,生成3-5句話的市場情緒總結
- **顯示方式**: 使用st.spinner顯示"正在分析市場情緒..."的載入動畫,完成後直接顯示AI生成的文字內容
- **AI分析重點**: 根據新聞標題、摘要、股票情緒、文章情緒、相關性和主題等資訊,提供簡短但全面的市場情緒狀態描述

#### 區域四:AI分析結果
- **標題**: "四、AI分析結果"
- **功能**: 整合股價數據和新聞情緒數據,使用OpenAI生成結構化的綜合分析報告
- **顯示方式**: 使用st.spinner顯示"正在生成投資建議..."的載入動畫,完成後使用markdown格式顯示分析內容
- **報告結構**: 
  - ## 分析總結
  - ## 媒體情緒觀察(依據標題和摘要簡單提出分析論述)
  - ## 綜合分析結果(短期與中長期)
- **分析數據來源**: 
  - 最近30天股價收盤價
  - 所有新聞的標題、摘要、情緒評分、相關性、主題資訊

### F-005: 基本資訊展示
