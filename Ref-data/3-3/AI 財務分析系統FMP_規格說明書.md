# AI 財務分析系統 規格說明書

> **用途說明**: 本規格說明書作為AI提示語使用，將此完整內容提交給AI，AI將根據規格生成完整的應用程式碼。

## 📋 系統概述

### 系統名稱
【Code Gym】AI 分析公司基本面應用

### 核心功能描述
建立一個基於網頁的財務分析應用，能夠：
1. 從Financial Modeling Prep API獲取完整的財務報表數據（損益表、資產負債表、現金流量表）
2. 進行四階段專業財務分析：Piotroski F-Score、Altman Z-Score、杜邦分析、現金流分析
3. 提供完整的財務數據視覺化和互動式圖表展示
4. 使用AI進行深度財務分析和投資評估解讀

### 技術架構要求
- **界面框架**: 使用 Streamlit 框架
- **數據來源**: Financial Modeling Prep API（三個獨立API端點）
- **AI 模型**: OpenAI
- **AI 模型版本**: o4-mini
- **視覺化工具**: 互動式圖表 (使用 Plotly Graph Objects 和 Plotly Express)
- **數據處理**: Pandas, NumPy
- **HTTP請求**: requests
- **日期處理**: datetime
- **部署方式**: 可直接在瀏覽器中運行

### 重要套件版本要求
- **OpenAI套件**: 使用 openai 版本 1.0.0 或以上的新API格式
- **禁用舊版API**: 絕對不可使用 ChatCompletion.create 舊格式
- **必須使用新格式**: 使用 OpenAI 客戶端初始化方式和 chat.completions.create 方法

**數據來源API端點**:
- 損益表API: https://financialmodelingprep.com/stable/income-statement?symbol={ticker}&apikey={api_key}
- 資產負債表API: https://financialmodelingprep.com/stable/balance-sheet-statement?symbol={ticker}&apikey={api_key}
- 現金流量表API: https://financialmodelingprep.com/stable/cash-flow-statement?symbol={ticker}&apikey={api_key}
- 企業價值API: https://financialmodelingprep.com/stable/enterprise-values?symbol={ticker}&apikey={api_key}
- 公司資料API: https://financialmodelingprep.com/stable/profile?symbol={ticker}&apikey={api_key}
- 關鍵指標API: https://financialmodelingprep.com/stable/key-metrics?symbol={ticker}&apikey={api_key}

**重要：FMP API欄位名稱標準化要求**
系統必須建立完整的欄位對應機制，將FMP API回傳的欄位名稱轉換為程式內部使用的標準名稱：

**損益表標準欄位對應**：
- FMP的 revenue 對應內部的 revenues（營收）
- FMP的 grossProfit 對應內部的 grossprofit（毛利）
- FMP的 operatingIncome 對應內部的 operatingincomeloss（營業利潤）
- FMP的 netIncome 對應內部的 netincomeloss（淨利潤）
- FMP的 interestExpense 對應內部的 interestexpensenonoperating（利息費用）
- FMP的 incomeBeforeTax 對應內部的 incomelossfromcontinuingoperationsbeforeincometaxes
- FMP的 weightedAverageShsOut 對應內部的 weightedaveragenumberofsharesoutstandingbasic

**資產負債表標準欄位對應**：
- FMP的 totalAssets 對應內部的 assets（總資產）
- FMP的 totalLiabilities 對應內部的 liabilities（總負債）
- FMP的 totalStockholdersEquity 對應內部的 stockholdersequity（股東權益）
- FMP的 totalCurrentAssets 對應內部的 assetscurrent（流動資產）
- FMP的 totalCurrentLiabilities 對應內部的 liabilitiescurrent（流動負債）
- FMP的 retainedEarnings 對應內部的 retainedearningsaccumulateddeficit（保留盈餘）
- FMP的 longTermDebt 對應內部的 longtermdebtnoncurrent（長期負債）

**現金流量表標準欄位對應**：
- FMP的 netCashProvidedByOperatingActivities 對應內部的 netcashprovidedbyusedinoperatingactivities（營運現金流）
- FMP的 operatingCashFlow 對應內部的 netcashprovidedbyusedinoperatingactivities（營運現金流-替代欄位）
- FMP的 netCashProvidedByInvestingActivities 對應內部的 netcashprovidedbyusedininvestingactivities（投資現金流）
- FMP的 netCashProvidedByFinancingActivities 對應內部的 netcashprovidedbyusedinfinancingactivities（融資現金流）
- FMP的 capitalExpenditure 對應內部的 paymentstoacquireproductiveassets（資本支出）

**現金流量表補充欄位**（可直接使用，無需計算）：
- FMP的 freeCashFlow：自由現金流（已計算值）
- FMP的 netChangeInCash：現金淨變動
- FMP的 cashAtEndOfPeriod：期末現金餘額
- FMP的 cashAtBeginningOfPeriod：期初現金餘額

**數據處理要求**: 
- **必須實施數據合併機制**: 將三個獨立API的數據合併為統一格式
- **必須標準化欄位名稱**: 將FMP API回傳的欄位名稱轉換為程式內部使用的標準名稱
- **必須建立日期索引**: 確保三個報表的數據能正確按日期匹配
- **必須處理缺失數據**: 提供預設值和錯誤處理機制

### F-003: 數據處理與計算
**處理目標**: 將原始財務數據轉換為四階段分析用的精確計算結果

**階段一：Piotroski F-Score 計算邏輯**
系統必須計算9項指標，每項以1分或0分評分，總分為0到9分：

**重要：F-Score計算基準**
- 系統必須計算**最新年度**相對於**前一年度**的F-Score
- 財務數據必須按日期降序排列（最新年度在前）
- 使用 financial_data[0] 作為當前年度，financial_data[1] 作為前一年度
- 如果財務數據少於2年，則無法進行年度比較分析

**獲利能力指標（4項）**：
1. **ROA正值檢查**: 計算最新年度ROA等於淨利潤除以總資產，如果大於0則得1分，否則0分
2. **營運現金流正值檢查**: 如果最新年度營運現金流大於0則得1分，否則0分
3. **ROA年增率檢查**: 比較最新年度ROA與前一年度ROA，如果最新年度較高則得1分，否則0分
4. **營運現金流品質檢查**: 比較最新年度營運現金流與淨利潤，如果營運現金流較高則得1分，否則0分

**槓桿與流動性指標（3項）**：
5. **長期負債比率改善檢查**: 計算長期負債除以總資產的比率，比較最新年度與前一年度，如果最新年度比率較低則得1分，否則0分
6. **流動比率改善檢查**: 計算流動資產除以流動負債的比率，比較最新年度與前一年度，如果最新年度比率較高則得1分，否則0分
7. **股份稀釋檢查**: 比較最新年度與前一年度的流通股數，如果最新年度未增加或減少則得1分，否則0分

**營運效率指標（2項）**：
8. **毛利率改善檢查**: 計算毛利除以營收的比率，比較最新年度與前一年度，如果最新年度比率較高則得1分，否則0分
9. **資產周轉率改善檢查**: 計算營收除以總資產的比率，比較最新年度與前一年度，如果最新年度比率較高則得1分，否則0分

**F-Score結果結構要求**：
系統必須回傳包含以下結構的結果：
- total_score: 總分（0-9分）
- profitability_scores: 獲利能力指標詳細結果（4項）
- leverage_scores: 槓桿與流動性指標詳細結果（3項）  
- efficiency_scores: 營運效率指標詳細結果（2項）
- 每項指標包含：score（得分）、description（說明）、相關數值（當前值、前期值等）

**階段二：Altman Z-Score 計算邏輯**
系統必須計算5個組成要素，使用公式 Z = 1.2×A + 1.4×B + 3.3×C + 0.6×D + 1.0×E：

**五個組成要素的計算**：
- **A項**：營運資本除以總資產，其中營運資本等於流動資產減去流動負債，再乘以1.2
- **B項**：保留盈餘除以總資產，再乘以1.4
- **C項**：EBIT除以總資產，其中EBIT等於營業利潤加上利息費用（重要：直接相加，不使用絕對值函數，保持利息費用的原始符號），再乘以3.3
- **D項**：市值除以總負債，再乘以0.6
- **E項**：營收除以總資產，再乘以1.0

**風險等級判斷**：
- Z-Score大於2.99為安全區域
- Z-Score介於1.81到2.99為灰色區域  
- Z-Score小於1.81為危險區域

**階段三：杜邦分析計算邏輯**
系統必須將ROE分解為三個因子：

**三因子分解計算**：
- **淨利率**：淨利潤除以營收
- **資產周轉率**：營收除以總資產
- **權益乘數**：總資產除以股東權益
- **ROE驗證**：淨利率乘以資產周轉率乘以權益乘數，結果應等於淨利潤除以股東權益

**趨勢分析要求**：
- 分析最近3年的三因子變化
- 計算各因子的年度變化量
- 識別ROE變化的主要驅動因子

**階段四：現金流分析計算邏輯**
系統必須計算現金流相關指標：

**關鍵指標計算**：
- **營運現金流品質比率**：營運現金流除以淨利潤
- **自由現金流**：營運現金流減去資本支出的絕對值（重要：使用絕對值確保資本支出為正值後再減去）
- **現金流結構分析**：計算營運、投資、融資三大現金流的金額和佔比

**品質評估標準**：
- 現金流品質比率大於等於1.2為優秀
- 現金流品質比率大於等於1.0為良好
- 現金流品質比率大於等於0.8為尚可
- 現金流品質比率小於0.8為需關注

### F-006: AI分析功能

**重要：OpenAI API 新版本實施要求**
系統必須使用OpenAI新版API格式，具體要求如下：
- **客戶端初始化**: 使用OpenAI類別創建客戶端實例，傳入API金鑰
- **API呼叫方式**: 使用客戶端的chat.completions.create方法
- **禁止使用舊版**: 絕對不可使用ChatCompletion.create舊格式，此格式在1.0.0版本後已停用
- **錯誤處理**: 包含完整的異常捕獲和友善的錯誤訊息

**AI分析目標**: 使用人工智慧對四階段財務分析結果進行專業解讀
**AI角色設定**: 專業的財務分析師，精通財報分析和投資評估

**AI提示語結構設計**:
- **系統訊息**: 設定AI為專業財務分析師角色
- **用戶提示語**: 包含已計算完成的四階段分析結果和完整財務數據
- **分析要求**: 要求AI基於計算結果進行解讀，而非重新計算
- **輸出格式**: 要求結構化的分析報告，包含表格和具體建議

**具體分析要求內容**:
1. **Piotroski F-Score 解讀**: 使用已計算的得分解讀各項指標的投資意義和業務狀況
2. **Altman Z-Score 風險評估**: 基於已計算的Z-Score值進行風險等級判斷和組成要素影響分析
3. **杜邦分析趨勢洞察**: 分析ROE三因子的趨勢變化和主要驅動力，發現財務效率變化
4. **現金流結構深度分析**: 分析現金流品質、組成變化、資本支出模式和獲利品質一致性
5. **綜合財務健康診斷**: 結合四項分析發現潛在風險或機會，提供整體評估

**綜合評估表格要求**:
要求AI輸出四階段評分總結表格，包含分析階段、評分狀態、評價、主要發現四個欄位

**分析結論要求**:
- **主要優勢**: 列出3到5個關鍵優勢
- **風險因素**: 列出需要關注的風險點
- **後續追蹤重點**: 投資後需要監控的關鍵指標
- **財報綜合評比**: 使用三大財報進行營運績效、財務結構、現金流量和總結分析

### F-008: 錯誤處理與用戶體驗

**數據驗證要求**:
