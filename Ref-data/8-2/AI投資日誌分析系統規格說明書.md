# AI投資日誌分析系統 規格說明書

> **用途說明**: 本規格說明書作為AI提示語使用,將此完整內容提交給AI,AI將根據規格生成完整的應用程式碼。

## 📋 系統概述

### 系統名稱
【Code Gym】AI投資日誌分析系統

### 核心功能描述
建立一個基於網頁的投資日誌分析工具,能夠:
1. **上傳並驗證投資日誌CSV檔案** - 自動檢查格式正確性與資料完整性
2. **即時計算投資績效** - 自動計算未實現與已實現損益、持倉狀況
3. **視覺化交易歷程** - 在K線圖上標記買賣點,清楚呈現每筆交易決策
4. **AI多維度客觀評估** - 從交易執行、決策品質、市場環境、心理因素進行深度分析
5. **提供具體改進建議** - 找出投資盲點,給予可執行的成長方向

### 教育核心理念
- **自我反思的力量**: 唯有透過不斷反思自己的錯誤和情緒,才能成為成熟的投資人
- **AI的私密性優勢**: AI可以客觀分析你的投資歷程,但不會分享給他人
- **過去照見未來**: 透過檢視過去的決策模式,幫助我們做出更理性的判斷
- **情緒與理性的平衡**: 誠實記錄交易時的心情,才能看見貪婪與恐懼如何影響決策

### 技術架構要求
- **界面框架**: 使用 Streamlit 框架
- **股價數據來源**: Financial Modeling Prep API (歷史股價、公司資料、財務指標)
- **新聞情緒數據**: Alpha Vantage API (可選,用於新聞情緒統計分析)
- **AI 模型**: OpenAI
- **AI 模型版本**: gpt-4o-mini
- **視覺化工具**: 互動式圖表 (使用 Plotly Graph Objects)
- **數據處理**: Pandas, NumPy
- **HTTP請求**: requests
- **日期處理**: datetime, timedelta
- **部署方式**: 可直接在瀏覽器中運行

### F-002: 數據獲取功能
**功能目標**: 整合多個數據源,建立完整分析資料集

**用戶操作**: 上傳投資日誌CSV後,系統自動獲取相關股票數據

**數據來源與說明**:

1. **投資日誌CSV檔案** (用戶上傳):
   - 必要欄位: Date(YYYY-MM-DD格式), Type(Buy/Sell), Symbol(股票代碼), Name(公司名稱), Price(交易價格), Quantity(數量), Reason(交易理由)
   - 需包含投資者的交易決策理由和當時心情
   
2. **Financial Modeling Prep API** (使用最新stable端點):
   - 歷史股價數據: `https://financialmodelingprep.com/stable/historical-price-eod/full?symbol={symbol}&apikey={api_key}`
   - 公司基本資料: `https://financialmodelingprep.com/stable/profile?symbol={symbol}&apikey={api_key}`
   - 關鍵財務指標: `https://financialmodelingprep.com/stable/key-metrics?symbol={symbol}&apikey={api_key}`
   - 財務比率: `https://financialmodelingprep.com/stable/ratios?symbol={symbol}&apikey={api_key}`
   - 當前報價: `https://financialmodelingprep.com/stable/quote?symbol={symbol}&apikey={api_key}`

3. **Alpha Vantage API** (可選):
   - 新聞情緒分析: `https://www.alphavantage.co/query?function=NEWS_SENTIMENT&tickers={symbol}`
   - 用於統計新聞正負面情緒比例,傳給AI作為分析參考

**數據範圍**:
- 股價數據: 從最早交易日期前120天開始
- 移動平均線: MA5, MA10, MA20, MA60
- 新聞數據: 近期新聞的情緒統計(看漲/中立/看跌比例)
- 財務指標: ROE, ROA, ROIC, 流動比率, 淨負債/EBITDA, 自由現金流殖利率

**資料處理要求**:
- 自動驗證CSV格式,提供詳細的錯誤和警告訊息
- 檢查API回應有效性,友善的錯誤提示
- 支援多檔股票的批次數據獲取

### F-003: 數據處理與計算
**處理目標**: 將投資日誌和市場數據整合,計算完整績效指標

**核心計算項目**:

1. **投資績效計算**:
   - 總投入成本: 所有買入交易的總金額
   - 已實現損益: 已賣出股票的實際獲利/虧損
   - 未實現損益: 目前持有股票的帳面獲利/虧損  
   - 總報酬率: (已實現損益 + 未實現損益) / 總投入成本
   - 當前投資組合市值: 持有股票數量 × 當前股價

2. **持倉分析計算**:
   - 各股票的平均成本價
   - 當前持有數量
   - 未實現損益金額和百分比
   - 持倉比重分配

3. **交易紀錄分析**:
   - 買入賣出次數統計
   - 每筆交易的報酬率(已賣出的交易)
   - 持有時間統計
   - 交易密集度分析

4. **技術指標計算**:
   - 移動平均線(MA5/10/20/60)
   - 交易點位與技術指標位置的關係

**數據整合要求**:
- 為每檔股票建立獨立的分析數據集
- 整合交易紀錄、股價數據、公司資料、財務指標、新聞情緒
- 計算股價變化的關鍵統計(期間最高/最低/起始/結束價格)
- 準備結構化的AI分析用數據

**品質要求**:
- 計算結果準確無誤,處理除零等異常情況
- 日期對齊正確,確保交易點標記在正確的K線上
- 妥善處理API數據缺失的情況

### F-004: 主要顯示區域設計
**頁籤結構**: 根據投資日誌中的股票數量動態創建頁籤

**投資組合總覽區域** (在頁籤上方):
1. **整體績效儀表板**: 使用3欄式配置展示關鍵績效指標
   - 第一欄: 總投入成本 (美金)
   - 第二欄: 投資組合市值 (美金,含變化率)
   - 第三欄: 總報酬率 (百分比,含金額)

2. **當前持倉狀況**: 表格展示所有持有股票
   - 欄位: 股票代碼、公司名稱、持有數量、平均成本、當前價格、未實現損益、報酬率
   - 排序: 按持有數量降冪排序

**個股分析頁籤** (每檔股票一個頁籤):
1. **股價走勢與交易紀錄圖** (雙子圖):
   - 上圖: K線圖 + 移動平均線(MA5/10/20/60) + 交易點標記
     - 買入點: 綠色向上三角形,懸停顯示交易詳情
     - 賣出點: 紅色向下三角形,懸停顯示交易詳情和獲利
   - 下圖: 成交量長條圖
   - 互動功能: 縮放、平移、懸停顯示完整交易資訊

2. **歷史交易詳情**: 可展開/收合的詳細表格
   - 欄位: 日期、類型(買入/賣出)、價格、數量、交易金額、理由、已實現損益(賣出時)
   - 排序: 按日期降冪排序

3. **公司基本資料**: 3欄式展示
   - 第一欄: 公司名稱、產業類別
   - 第二欄: 當前股價、52週區間  
   - 第三欄: 交易所、市值

4. **關鍵財務指標**: 3欄式展示
   - 第一欄: ROE、ROA
   - 第二欄: 淨負債/EBITDA、流動比率
   - 第三欄: 自由現金流殖利率、ROIC

5. **AI深度分析報告** (如有啟用):
   - 完整的Markdown格式分析報告
   - 包含五大分析維度和綜合建議

### F-005: 基本資訊展示
**展示方式**: 使用 st.columns(3) 展示投資組合整體績效

**展示內容**:
- 第一欄: 總投入成本 (所有買入交易的總金額)
- 第二欄: 投資組合市值 (當前持有股票的總市值,含變化率delta)
- 第三欄: 總報酬率 (含已實現和未實現損益的總報酬,含金額delta)

**持倉狀況表格**:
- 展示當前所有持有的股票
- 包含欄位: 代碼、名稱、數量、成本、價格、損益、報酬率
- 使用條件格式標示獲利(綠色)和虧損(紅色)

**數值處理**:
- 金額格式化: 使用千分位逗號,保留2位小數
- 百分比顯示: 保留2位小數,加上%符號
- Delta變化值: 使用st.metric的delta參數,自動顯示綠色或紅色箭頭
